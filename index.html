HTML Tags

<b> and <strong>

Both of tags will make text look bold. The second tag also describes a higher text importance than the first. Also, according to the HTML 5 specification, the <b> tag should only be used as a LAST resort when no other tag is more appropriate.

HTML : <b>bold text</b>

Result : bold text

<i> and <em>

Both tags will make a text look italic. The second tag also describes a higher emphasis than the first. Also, according to the HTML 5 specification, the <i> tag should be only used as a LAST resort when no other tag is more appropriate.

HTML: <i>italic text</i>

Result: italic text

<br>

This tag inserts a single line break. This line serves as the 'enter' key for the webpage, as it does on a word document. Also, this is an empty self-closing tag which means that it has no need for the extra closing tag.

Example : <br /> Text

Result:
Text

<hr>

This element is used to separate content (or define a change) in the HTML page. This tag puts a nice line across the whole page within the box model of the parent element. This tag is mainly used as a separator for different articles.

Example : <hr /> Text

Results:

Text

<p>

This is the paragraph tag, widely used for containing a block of a text. This paragraph has some default left margin from the user agent CSS.

HTML: <p>Text</p>

Result:

Text

<ins>

This tag is to be used to notify users that some new text that has been additionally inserted into some other text. By default, the browsers display an underline to make the text stand out. As far as I have seen, this reason this the tag is often not seen compared to its counterpart <del>.

HTML: <ins>Text<ins>

Result: Text

<s>

This tag is similar to the <del> tag, The difference is that text is not relevant anymore, yet not deleted. This tag can be used by online shops to highlight a price discount.

HTML: Price: <s>£199</s>

Result: Price: £199

<del>

This tag is to be used on text that is to be displayed as deleted, but kept as a reference. This tag is usually used in blogs websites where the incorrect text is marked, but not deleted. Usually to show that the text was wrong, and corrected. The <s> and <del> tags display identically in the browser, the extra that they do is give more meanings to the screen readers which is important for serious websites.

HTML: It was my <del>worst</del> idea.

Result: It was my worst best idea.

<mark>

This tag is to be used to define and highlight parts of your text in a way a marker would on a piece of paper.

HTML: <mark>Text</mark>

Result: Text

<sup>

This tag defines superscript text. Superscript text appears half a character above the normal line and is rendered in a smaller font.

HTML: 5<sup>th<sup/>

Result: 5th

<sub>

This tag defines subscript text. Subscript text appears half a character below the normal line and is rendered in a smaller font.

HTML: H <sub>2<sub/>O

Result: H2O

<cite>

This tag is used to define text that you would want to reference. In HTML5 this should refer to the text itself and not the author.

HTML: <cite>Text</cite>

Result: Text

<abbr>

This is the new acronym tag and it's used for abbreviations. Basically, if you have a shortened term like 'NSA' you can have this text around this tag, and when the user hovers over it he will get the full term. This is also used by screen readers.

HTML: <abbr>Text</abbr>

Result:(FIX-FIX-FIX!) Text

<dfn>

This tag is used to explain new terminologies. They call it the defining instance of a new term. So if you have explained something like for example a hedge fund you would use this tag around that word. This will be used by screen readers, most browsers would not do anything to the text itself, some would make the text italic.

HTML: <dfn>Text</dfn>

Result: Text

<backquote>

This tag is used to define and reference a full paragraph quote. This tag is very similar to the <q> tag but where the <q> is for short quotes, this tag is for full paragraph quote.

HTML:


<backquote>
<p>
You've gotta dance like there's nobody <br>
watching, Love like you'll never be hurt, Sing like <br>
there's nobody listening, And live like it's heaven <br>
on earth. – William W. Purkey
</p>
</backquote>

Result:


You've gotta dance like there's nobody
watching, Love like you'll never be hurt, Sing like
there's nobody listening, And live like it's heaven
on earth. – William W. Purkey

<address>

This is the tag used for addresses, it's pretty much self-explanatory. just use other tags like </br>, </div> or </p> with a indent class to properly display your address.

HTML:


<address>
Business Name <br>
The Parkside Building <br>
5 Cardigan St <br>
Birmingham <br>
B4 7BD <br>
<address>

Result:


Business Name
The Parkside Building
5 Cardigan St
Birmingham
B4 7BD

CSS Grid

The 1st example shows somewhat default look of divs within a div, without any special CSS grid layout coding.

CSS:

// The following visual styles are applied to all elements with .grid class.
// Only set black text color if you use white or light gray background.

.grid {
  color: black;
}

// Set border, background, padding and border-radius for all CSS grid elements.

.grid > div {
 border: 1px solid #ccc;
 background: #eee;
 padding: 1em;
 border‐radius: 5px;
}

// Set different background to all odd CSS grid elements to make them stand out.

.grid > div:nth‐child(odd) {
 background: #ddd;
}

// Set all elements with class grid to be displayed as a CSS grid.
// Then add a gap of 10px which would be applied to row and colums.

.grid {
  display: grid;
  grid‐gap: 10px;
}

In this 5th example, you can see the minmax height subfunction used with parameters of 100px and auto. This means that the minimum height of a box will be 100px and if the content overflows the height of the box will adjust itself automatically, as it can be seen below.

You can also see nested divs with their own grid system. Visually they have a gap value, also some padding and border. You can also see that they can be aligned any way you want using the appropriate coding.

To view this example, please open this page on a device with a larger screen.

CSS:

// Set layout as 5 equaly spaced out columns utilizing repeat function.
// Set the minimum hieght of each nested div as 10px
// and to be expandable if it needs to be.

.grid_5 {
 grid‐template‐columns: repeat(5, 1fr);
 gird‐auto‐rows: minmax(10px, auto);
}

// Set the nested divs to be displayed as CSS grid.
// Set layout as 3 equaly spaced out columns utilizing repeat function.
// Set static height of 50px for each of the nested divs.

.nestedDiv {
 display: grid;
 grid‐template‐columns: repeat(3, 1fr);
 grid‐auto‐rows: 50px;
}

// Add border and padding to the nested divs to differentiate them from the parent div.

.nestedDiv > div {
 border: 1px dotted black;
 padding:left: 5px;
}

// Div 1 - Doesn't have any special coding.

// Div 2 to 7 - The CSS coding aligns items to particular space of their layout boundaries.
// For example, justify-items is capable of aligning the items vertically and align-items does the opposite, it aligns items horizontally.
// Also, you can use 3 different parameters on both, those include start, center, and end.

.nestedDiv2 {
 justify‐items: start;
}

.nestedDiv3 {
 justify‐items: center;
}

.nestedDiv4 {
 justify‐items: end;
}

.nestedDiv5 {
 align‐items: start;
}

.nestedDiv6 {
 align‐items: center;
}

.nestedDiv7 {
 align‐items: end;
}

// The final 3 examples use classes on the parent divs, to move the divs to a particular spot within their layout boundary.
// No nested divs have been used here.

.exampleDiv1 {
 align‐items: start;
}

.exampleDiv2 {
 align‐items: center;
}

.exampleDiv3 {
 align‐items: end;
}

As you can see below in this 6th example. This is perhaps the most powerful feature of the CSS Grid system. This separates it from flexbox and floats. As you can see you can stretch individual boxes in pretty much any direction within the box model. All that being said as you can see the boxes automatically resize each other to keep within their boundaries.

To view this example, please open this page on a device with a larger screen.

CSS:

// For this final example, we set the look of the gird. We use fractional units as 1-2-1.
// We also set the minimum height of the divs as 100px, which is expandable if it needs to be.

.grid_6 {
 grid‐template‐columns: 1fr 2fr 1fr;
 grid‐auto‐rows: minmax(100px, auto);
}

// This is perhaps the most sophisticated feature of the CSS grid. The ability to overlap.
// This can be done by imagining the whole layout as a bunch of different lines.
// To see an example of these lines please click here.
// Once you imagine your layout as having those lines, you can code different objects to overlap their boundaries and go into other objects while having those overlapped objects simply be moved automatically.
// To see it clearly please look at the below example and the picture from the above link.

.box_1 {
 grid‐column: 1/3;
 grid‐row: 1/3;
}

.box_2 {
 grid‐row: 1/3;
}

.box_3 {
 grid‐column: 1;
 grid‐row: 2/4;
}

.box_3 {
 grid‐column: 1;
 grid‐row: 2/4;
}

.box_4 {
 grid‐column: 2/4;
}
